## How to keep this fresh

The docs are a **point-in-time snapshot**, not self-healing. Rerun the prompt `FIX_100_FULL_ECOSYSTEM_MAP.md` after:
- Any new feature landing (updates the matrix)
- New Worker / D1 / R2 / KV resources being provisioned (updates inventory)
- Any schema change on bound DBs (updates data model)
- Quarterly as a health check

The **auto-generated** counterparts stay current on their own:
- [`docs/API.md`](../../API.md) — regenerated by CI workflow `docs-check.yml` on every PR
- [`docs/MIGRATIONS.md`](../../MIGRATIONS.md) — migration rollback rules stay valid as long as the convention is enforced
- [`docs/FIX_LOG.md`](../../FIX_LOG.md) — manually updated per fix PR
